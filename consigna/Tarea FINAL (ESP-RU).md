## Explicación del trabajo ***(Пояснение к работе)***

Finalizar el desarrollo de la aplicación web planificada durante la cursada, con el proyecto resuelto.

## Modalidad ***(Формат разработки)***

Desarrollar un sitio web con la temática definida durante la cursada y que incluya:  
*→ Разработать веб-сайт с тематикой, определённой в течение курса, который включает:*

- Gestión de la información (CRUD)  
    *→ Управление информацией (CRUD)*
    
- Login y logout de usuarios.  
    *→ Вход и выход пользователей.*
    
- Tres o más roles de usuarios.  
    *→ Три или более ролей пользователей.*
    
- Restricciones de acceso a los distintos tipos de usuarios según corresponda.  
    *→ Ограничения доступа для различных типов пользователей в соответствии с их ролями.*
    

## Puntos a tener en cuenta ***(Пункты, которые следует учитывать)***

- La información ingresada por el usuario tanto en las URL, como formularios debe sanearse y validarse correctamente.  
    *→ Информация, введённая пользователем как в URL, так и в формах, должна быть правильно очищена и валидирована.*
    
- Los registros deben mostrarse paginados.  
    *→ Записи должны отображаться с пагинацией.*
    
- Toda la información debe gestionarse a través de una interfaz de usuario, excepto casos puntuales como por ejemplo los roles.  
    *→ Вся информация должна управляться через пользовательский интерфейс, за исключением отдельных случаев, таких как, например, роли.*
    
- Proteger correctamente las páginas de personas sin acceso a las mismas.  
    *→ Правильно защитить страницы от лиц, не имеющих к ним доступа.*  
    Por ejemplo si es una acción que sólo puede realizar alguien con rol administrador, otro usuario con otro rol no debería entrar ni siquiera haciendo copy-paste de la URL.  
    *→ Например, если это действие, которое может выполнять только кто-то с ролью администратора, другой пользователь с другой ролью не должен иметь возможности войти даже при копировании URL.*
    
- El sitio web debe tener un diseño bien logrado.  
    *→ Веб-сайт должен иметь хорошо проработанный дизайн.*  
    También debe ser propio del proyecto y no una plantilla por default de la tecnología utilizada.  
    *→ Также он должен быть собственным для проекта, а не шаблоном по умолчанию используемой технологии.*
    
- Se debe implementar el patrón MVC o cualquier otro, mientras el código se separe correctamente.  
    *→ Необходимо реализовать паттерн MVC или любой другой, при условии правильного разделения кода.*
    

Además el trabajo deberá tener consigo un documento, de estilo manual (word, pdf, video, etc) explicando las distintas funcionalidades y aclarando los permisos que se deben tener para cada una.  
*→ Кроме того, работа должна сопровождаться документом в стиле руководства (word, pdf, видео и т.д.), объясняющим различные функциональности и уточняющим разрешения, необходимые для каждой из них.*

Éste debe venir acompañado de capturas de pantalla que ayuden a entender el trabajo a nivel visual.  
*→ Он должен сопровождаться снимками экрана, которые помогут понять работу на визуальном уровне.*

**Importante**: el día de la exposición, el grupo debe tener cargado un set de datos para probar el correcto funcionamiento del sistema.  
*→ Важно: в день презентации группа должна иметь загруженный набор данных для проверки правильного функционирования системы.*
